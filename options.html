Instructions:

Read the Authoritative guide to CORS (Cross-Origin Resource Sharing) for REST APIs first to understand how it works and common pitfalls.

This plugin allows you to send and debug cross-domain requests without receiving Cross-Origin Errors. The browser automatically sets the CORS response headers, but you can override it via this plugin.

Because wildcard is not allowed on Access-Control-Allow-Origin when Access-Control-Allow-Credentials is set to true and since wildcard is not implemented yet for Access-Control-Allow-[Headers, Methods] in current browsers, this plugin will do the following:
(More info)

  if (Access-Control-Allow-Origin == '*' && Access-Control-Allow-Credentials == 'true') {
    Access-Control-Allow-Origin = <The Origin from Request Origin Header>
  }

  if (Access-Control-Allow-Headers == '*') {
    Access-Control-Allow-Headers = <Comma Separated list of all request headers
      including any existing Access-Control-Allow-Headers. You may have to check>
  }

  if (Access-Control-Allow-Methods == '*') {
    Access-Control-Allow-Methods = 'GET, HEAD, POST, PUT, DELETE, CONNECT, OPTIONS, TRACE, PATCH'
  }

For security, it is recommended to disable plugin when done with debugging. This simple tool is provided without warranty of any kind.
